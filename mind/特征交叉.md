# MIND‑small 最新代码分析与显式特征交叉方案

## 1 新代码特征与模型分析

### 1.1 新闻特征构建
新闻样本被转换为多个离散与连续特征：

- **类别（category）与子类别（subcategory）**：对原表中的 `category` 与 `subcategory` 字段去重并建立索引映射。
- **标题与摘要 token**：分别从 `title_tokens` 和 `abstract_tokens` 列解析出词列表，生成词典并对每条新闻截取固定长度（标题默认 24 个词，摘要默认 48 个词）。
- **实体 ID 列表**：从 `entities` 字段解析实体 ID，构建实体词典并截取长度（默认为 5）。
- 上述离散索引分别映射为张量：`news_category_ids`、`news_subcategory_ids`、`news_title_token_ids`、`news_abstract_token_ids`、`news_entity_ids` 等。

相较上一版，新增了**摘要 token**和**实体 ID**特征，并统一采用 `feature_schema_version=3` 以确保缓存兼容性。

### 1.2 编码模型

新版代码包含两个新闻编码器：

1. **BaselineNewsEncoder (`modelbaseline.py`)**：将类别、子类别、标题均值、摘要均值（可选实体均值）嵌入拼接后通过线性层映射到统一的 `embedding_dim`。用户编码器仅对历史新闻向量做简单平均，打分模块将用户向量与候选新闻向量做乘积、差值与点积等显式交叉，再通过 MLP 输出点击概率。
2. **FeatureNewsEncoder (`modelfeature.py`)**：在新闻内部引入位置编码和 Transformer 自注意力对标题与摘要 token 进行上下文建模。可选实体通过平均池化引入。该模型使用 **Multi‑Head Target Attention** (MHTA) 对每个候选新闻动态聚合用户历史：候选向量作为查询，历史新闻向量为键和值，通过多头注意力获得针对候选的用户表示并结合残差和前馈网络更新。打分器仍然对用户表示与候选向量做乘法、绝对差和点积显式交叉。

### 1.3 特征提升机会

- **数据维度**：MIND‑small 中新闻条目包含丰富的文本和实体信息，但 baseline 仅对标题做均值池化，忽略了摘要和实体信息。新版代码增加摘要和实体特征，可进一步挖掘细粒度语义。
- **交互建模**：现有打分器只针对整体用户向量和候选新闻向量做简单显式交叉，没有捕捉新闻内部各字段之间的互相作用，也未考虑候选新闻与用户历史分布之间的显式交叉。

针对这些不足，可以在新闻内部和用户‑候选层面引入显式交叉特征，然后通过 SENet 动态学习各交叉特征的重要性。

## 2 显式特征交叉方案

### 2.1 新闻内部交叉特征

将新闻的不同字段视为独立的向量（经 embedding 或 Transformer 得到）。设类别向量为 cc、子类别向量为 ss、标题表示为 tt、摘要表示为 aa、实体表示为 ee（若没有实体，则省略）。在编码新闻时，对这些向量先用线性层投影到统一维度 DD（例如 128），然后计算以下显式交叉特征：

|交叉对|计算方式|目的|
|---|---|---|
|**类别×子类别 (c⊗s)**|对投影后的 cc 与 ss 逐元素相乘 c⊙sc⊙s 或做差 (|c - s|
|**类别×标题 (c⊗t)**|将类别向量通过线性层扩展到 DD，与标题向量做 Hadamard 乘积；或直接拼接后用线性层学习交互|表示某一类别下哪些关键词重要，比如体育类新闻关注“NBA”这个词|
|**类别×摘要 (c⊗a)**|类似于类别×标题，用于捕捉类别与摘要词语的关联|摘要提供更多背景信息，帮助模型理解类别内不同风格的新闻|
|**类别×实体 (c⊗e)**|类别向量与实体向量逐元素相乘，再用线性层映射|描述某一类别常见的实体，如财经新闻常出现“Nasdaq”|
|**子类别×标题 (s⊗t)**|将子类别向量扩展后与标题向量交叉|提供比类别更细的主题与关键词组合|
|**子类别×摘要 (s⊗a)**|与上类似||
|**子类别×实体 (s⊗e)**|与上类似||
|**标题×摘要 (t⊗a)**|将标题与摘要的语义向量做元素乘或余弦相似度，并用 MLP 投影|捕捉标题和摘要之间的一致性和互补性，提高语义表达|
|**标题×实体 (t⊗e)**|将标题语义与实体均值向量做元素乘，或通过 bilinear 函数建模交互|体现标题关键词与实体的关系，如“乔布斯”这一实体对应的标题词|
|**摘要×实体 (a⊗e)**|类似于标题×实体||

> **实现建议：** 为了控制维度和计算量，可对类别、子类别、实体等离散特征先通过小尺寸 embedding（如 32/64 维）再投影到 DD。对文本向量（标题/摘要）使用共享或独立的投影层。交叉操作可选择 **逐元素乘**、**绝对差** 或 **拼接后线性层**；FiBiNet 也提供了 **双线性交互**，可以用两个线性变换 W1,W2W1​,W2​ 分别作用于两个特征后再逐元素相乘，从而获得更丰富的交互。计算得到的每个交叉向量再经 SENet（或轻量级的 Squeeze-Excitation 网络）动态调整权重。

### 2.2 新闻与用户历史显式交叉

基于 Multi‑Head Target Attention，候选新闻 nn 得到一个针对该候选的用户表示 unun​。除了将 unun​ 与候选整体向量做乘积、差值和点积，还可以构造更细粒度的交叉特征，捕捉候选新闻内部特征与用户历史偏好的匹配：

| 解释                                                         |
| ---------------------------------------------------------- |
| 衡量用户对该类别的兴趣强度，解决用户冷启动或长尾类别问题                               |
|                                                            |
| 描述用户关注的实体与候选新闻实体的一致性，例如用户经常阅读“特斯拉”相关新闻，则含有“Tesla”的候选更可能被点击 |
| 捕捉用户对特定关键词或主题的偏好，比如用户经常看关于“AI”的文章，则含有相关词的标题会受到强调           |
| 建模用户对新闻时效性的偏好（例如倾向于阅读最新新闻），时间差值可以与类别/实体进一步交叉               |

这些交叉特征可在构造候选‑用户匹配向量时与原本的 unun​ 和候选新闻向量一起送入 SENet 进行加权，从而补充现有打分器的简单交互。

### 2.3 离散 ID 级交叉特征

对于离散的类别和子类别，还可以在索引级别构造新的**组合 ID**，例如将 `(category, subcategory)` 拼接为一个新标识符，使模型直接学习类别‑子类别组合的嵌入。这相当于将 `category_id * N_subcategory + subcategory_id` 当作新的索引列。由于 MIND 数据集的类别（约 18 类）和子类别数量有限（数百个），组合数量在几千级别，可用一个独立 embedding 学习其表征。类似地，也可以根据业务需求构建 `(subcategory, entity)`、`(category, entity)` 组合，但应控制组合的规模。通过组合 ID，可以捕捉在某一类别和细分领域中频繁共现的实体或词语。

### 2.4 集成到 SENet 的建议

显式交叉会产生多个向量通道，采用 SENet 能够为每个通道学习重要性权重并进行动态重标定。具体步骤如下：

1. **特征投影**：将所有基本特征和交叉特征统一投影到相同维度 DD。
2. **拼接**：将这些向量沿 channel 维度堆叠形成张量 F∈RN×DF∈RN×D，其中 NN 是基本特征和交叉特征数目。
3. **Squeeze**：对每个通道做全局平均池化获得通道描述子；
4. **Excitation**：通过两层全连接（减小再扩张维度）学习权重向量 s∈RNs∈RN，并用 sigmoid 激活；
5. **重标定**：将 FF 各通道与权重 ss 相乘后求和，得到最终的新闻表示输入后续模型。

这种方式能让模型自动关注最有用的交叉模式并抑制噪声特征。对于候选‑用户交叉，类似地先构造交叉特征矩阵，再使用 SENet 对通道权重进行调制，然后送入打分器或者与原有交互特征拼接。

## 3 实现与实验建议

1. **特征选择与归一化**：优先选择对新闻和用户点击行为影响显著的交叉对，例如 `(类别, 子类别)`、`(类别, 实体)`、`(标题, 摘要)` 和 `(候选实体, 历史实体)`，避免组合过多导致计算量和过拟合。可以先在训练集上通过统计分析或互信息评估挑选交叉组合。对计数型交叉（如时间差分桶）可做归一化或使用 embedding 分桶。
2. **代码集成位置**：对于新闻内部交叉，可在 `FeatureNewsEncoder.forward` 中，完成标题/摘要/类别/实体向量的计算后加入交叉模块，然后用 SENet 压缩输出与原始特征相同维度；对于候选‑用户交叉，可在打分模块前将交叉特征与现有的 `mul_feat`、`abs_diff_feat`、`dot_feat` 拼接，再引入 SENet 或额外 MLP。
3. **实验对比**：可分别在 baseline 模型和 feature 模型中加入显式交叉 + SENet，与原始模型进行 AUC、MRR、nDCG 等指标比较。注意在 MHTA 中加入大量交叉特征可能增加计算开销，可使用小批量测试。还可以尝试 ListNet 或 ListMLE 等列表式损失，以验证交叉特征对排序效果的提升。
4. **冷启动与解释性**：离散 ID 级的交叉（例如 `(category, subcategory)`）既提高了冷启动新闻的表征能力，也方便业务人员理解模型偏好（如“财经‑股票”组合权重高”）。

通过以上显式交叉方案，可以充分挖掘 MIND‑small 数据中的类别、文本、实体及用户历史信息之间的关系，再结合 SENet 的动态加权能力，为模型提供更强的表示能力和更好的解释性。

报告中指出，MIND‑small 每条新闻现有的类别、子类别、标题 token、摘要 token 和实体 ID 都已经通过索引方式编码并映射为张量；在 `FeatureNewsEncoder` 中，通过位置编码和 Transformer 自注意力获得标题和摘要的语义表示。此外，模型的打分器只利用了用户与候选整体向量的简单乘积、差值和点积，未充分挖掘这些字段之间的交互潜力。

报告详细给出了如下交叉建议：

1. **新闻内部交叉**：将类别、子类别、标题、摘要、实体各向量投影到统一维度后，计算逐元素乘积或差值，如类别×子类别、类别×标题、标题×摘要等。每种交叉向量再通过 MLP 压缩并由 SENet 动态加权。
2. **候选与用户历史交叉**：利用 MHTA 为每个候选生成用户表示后，再构造候选类别×用户历史类别分布、候选实体×用户历史实体向量、候选文本×用户历史文本语义、候选新闻 recency×用户历史 recency 等交叉特征，这些特征可在打分模块前送入 SENet。
3. **离散 ID 级组合**：对 `(category, subcategory)` 等离散对建立新的组合索引并学习嵌入，捕捉细粒度主题组合；可根据业务需要扩展到 `(subcategory, entity)` 等组合，但需控制组合规模。

方案还给出了在代码中集成交叉特征的建议，强调应先挑选高价值的交叉对、在 `FeatureNewsEncoder.forward` 或打分模块前生成交叉特征并使用 SENet 调制通道权重，以及结合不同损失函数验证效果。
