# 改进思路：
**改 PhysResQuadModel 里的 residual head**
1. **借 Transolver 的“按物理分块/分 token 再交互”思想，做一个轻量的“物理分组 + cross-attention”残差网络；**
- 而 benchmark 论文自己就说了：平动（position / linear velocity）更容易：转动（orientation / angular velocity）更难；难点主要在 torque / motor slow dynamics。把状态按物理意义拆成 2～3 个 token：token A：平动块 [p, v]，token B：转动块 [r_log, ω]， token C：控制块 u_t 或控制序列编码 。然后每个块先各自编码，再做一层很小的 cross-attention / gated mixing，最后分别输出 Δx_trans 和 Δx_rot。

2. **借 Koopa 的“慢-快/时不变-时变分解”思想，做一个轻量的 slow-fast 分支；**
3. **先把 baseline repo 里现成但没用上的旋转几何损失用起来。把旋转损失从普通 MSE 换成 repo 里已经写好的** **WeightedGeodesicLoss**。
4. 受 zero-shot/foundation model 在动力系统预测中的启发，我们为 nano-drone system identification 设计了一个 **foundation-style pretraining pipeline**：先在大量滑窗轨迹上进行控制条件的自监督/监督预训练，再在官方 benchmark 上微调；可选地引入 Chronos-2 作为 teacher 做蒸馏正则。

**LSTM 明显不如 Physics 和 Phys+Res，主要问题是 drift；而旋转动力学尤其难，因为标准二次电机模型抓不住一个慢的低频 actuation 成分。** 论文还直接说了，后续要么需要**更强的 temporal model with longer context**，要么需要**更好的 rotor/motor physics**。这句话其实正好给了你改网络的论文动机。
更重要的是，train/losses.py 里其实已经有一个 **WeightedGeodesicLoss**，它对位置/速度/角速度做 MSE，对旋转用 SO(3) geodesic loss；但 train_lstm.py 实际用的是更简单的 **WeightedMSELoss**。这对你是个非常高性价比的突破口。



## pro改进

围绕“电机/执行器慢时变效应导致的旋转预测困难”做一个轻量时序增强的 hybrid model。** 论文本身已经明确：位置和线速度相对更容易拟合，真正难的是旋转动力学；作者还指出标准二次电机模型漏掉了低频 actuation dynamics。对应到代码里，PhysQuadModel 仍然是瞬时 u^2 → T, τ 映射，而且把 omega_dot 直接乘成了 0；QuadLSTM 也是用 x0 初始化后只沿 u_seq 展开。

**Delay-aware Phys+Res**
**核心想法**：在物理分支前面加一个非常轻量的“电机记忆/滞后模块”，不要再把当前时刻电机转速直接当成立即生效。![[Pasted image 20260328214812.png]]
其中 α 可以是 4 个电机各自一个参数，也可以先用 1 个共享参数。然后**物理模型用 u_eff 而不是 u_t** 去做 motor_to_phys()，残差网络则输入 [x_t, u_t, u_eff, u_t-u_eff]。这正好对准论文说的“低频 actuation 未建模”这个瓶颈，也正好对应当前代码里 motor_to_phys() 是瞬时映射、omega_dot 又被置零的局限。
